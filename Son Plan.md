# ErrorLife — Kapsamlı Mimari Revizyon, UX Düzeltmeleri ve Yeni Özellikler

## Context

Kullanıcı 4 alanda kritik düzeltme/ekleme istiyor:

1. **Web modal hiyerarşisi bozuk** — `web/src/app/@modal/(.)post/[id]` intercepting route düzgün çalışıyor ama: (a) modal `router.back()` ile kapatıldığı için tarayıcı history kirleniyor; (b) modal içinden profil linki tıklayınca modal DOM'da kalıyor (full-page nav ile temizlenene kadar overlay/scroll lock devam ediyor); (c) yorum yazarına tıklama da aynı sorunu doğuruyor.
2. **Mobil "Şifremi Unuttum" akışı yok** — `mobile/app/(auth)/` altında sadece `login.tsx` ve `register.tsx` var. Web tarafı (`web/src/app/(auth)/forgot-password/`) hazır; mail linki `${origin}/auth/callback?next=/reset-password` biçiminde web'e dönüyor.
3. **Mobil Bildirimler tab'ı yok** — `mobile/app/(tabs)/_layout.tsx` 4 sekme tanımlıyor (index/jobs/messages/profile). Web'de tüm logic hazır: 3 RPC (`get_interaction_notifications`, `get_follow_notifications`, `get_message_notifications`), tipler `packages/shared/src/types.ts`'de.
4. **Mimari/güvenlik denetimi** — `bookmarks` tablosunda SELECT `using(true)` (gizlilik açığı: kim neyi kaydetmiş herkes görebilir). Web ve mobil tip dosyaları (`web/src/types/database.ts`, `mobile/src/types/database.ts`) `packages/shared`'daki tipleri tamamen duplicate ediyor; `web/src/lib/schemas.ts` shared zod schema'larını duplicate ediyor; `mobile/app/jobs/[id]/index.tsx`'te local `Bid` type tanımı var.

Hedef: Bug fix + yeni özellikler + duplicate temizlik. Database type'larını dokunmadan bırakıyoruz (Supabase autogenerated, ayrı iş); diğer hardcoded duplicate'ler shared'a yönlendirilecek.

## Adım 1: Web Modal & Navigation Fix (KRİTİK)

### Sorunun anatomisi

| Senaryo | Mevcut davranış | Root cause |
|---|---|---|
| Bildirim → post tıkla | `router.push('/post/x')` → intercept çalışıyor, modal açılıyor ✓ | Çalışıyor |
| Modal kapatma çarpısı | `router.back()` → `/notifications`'a döner | Doğru ama modal içinde profile linki tıklandıysa history zaten ileri gitmiş, `back()` yanlış yere atıyor |
| Modal içinden profil linki | `<Link href="/profile/x">` soft nav → URL değişir, ama modal overlay DOM'dan silinmiyor (parallel slot remount edilmiyor); body `overflow:hidden` kalıyor | `PostModal`'ın `useEffect` cleanup'ı sadece unmount'ta çalışıyor; soft nav → @modal slot'u remount tetiklemiyor |

### Çözüm

**Dosya:** `web/src/app/_components/post-modal.tsx`

`router.back()`'i akıllandır: Eğer önceki history girdisi mevcut origin'de değilse veya history boşsa, `router.push('/')` ile fallback yap. Daha önemlisi: `usePathname` izle — pathname `/post/[id]` dışına çıktığında modal'ı **manuel olarak kapat** (state ile gizle ve cleanup'ı hemen çalıştır).

```tsx
"use client";
import { useEffect, useCallback, useState } from "react";
import { useRouter, usePathname } from "next/navigation";
import { X } from "lucide-react";

export default function PostModal({ children }: { children: React.ReactNode }) {
  const router = useRouter();
  const pathname = usePathname();
  const [open, setOpen] = useState(true);

  // Modal içinden farklı bir route'a soft-nav olduysa modal'ı kapat
  useEffect(() => {
    if (!pathname?.startsWith("/post/")) setOpen(false);
  }, [pathname]);

  const handleClose = useCallback(() => {
    setOpen(false);
    // history'de güvenli bir önceki giriş varsa back, yoksa ana sayfaya
    if (window.history.length > 1 && document.referrer.includes(window.location.origin)) {
      router.back();
    } else {
      router.push("/");
    }
  }, [router]);

  useEffect(() => {
    if (!open) return;
    document.body.style.overflow = "hidden";
    const onKey = (e: KeyboardEvent) => e.key === "Escape" && handleClose();
    document.addEventListener("keydown", onKey);
    return () => {
      document.body.style.overflow = "";
      document.removeEventListener("keydown", onKey);
    };
  }, [open, handleClose]);

  if (!open) return null;
  // ... mevcut JSX (backdrop + X button)
}
```

**Dosya:** `web/src/app/_components/post-detail-content.tsx` (satır 46-78)
**Dosya:** `web/src/app/_components/comment-form.tsx` (satır 150-182)

Modal içinde profil linki tıklandığında, modal hemen kapanmalı — bu otomatik oluyor (yukarıdaki `usePathname` izleme), ek değişiklik gerekmez.

**Doğrulama:**
- Bildirim → post modal aç → X tıkla → bildirimlere geri dön ✓
- Bildirim → post modal aç → modal içinde author profiline tıkla → modal kapanır, profil sayfası açılır, sidebar normal ✓
- Profil → post tıkla → post modal açılır → modal içindeki yorum yazarına tıkla → modal kapanır, yorum yazarının profili açılır ✓
- Direkt URL `/post/x` aç → modal değil, standalone sayfa açılır ✓

## Adım 2: Mobil "Şifremi Unuttum" Sayfası

### Yapı

**Yeni dosya:** `mobile/app/(auth)/forgot-password.tsx`

Login/register pattern'ine sadık kal: `KeyboardAvoidingView` + `TextInput` (NativeWind sınıfları) + `TouchableOpacity` button + `Alert.alert` ile hata/başarı.

```tsx
// Pseudocode
const WEB_URL = process.env.EXPO_PUBLIC_WEB_URL ?? "http://localhost:3000"; // TODO: prod URL'i .env'e ekle

const handleReset = async () => {
  if (!email) return Alert.alert("Hata", "E-posta gerekli.");
  setLoading(true);
  const { error } = await supabase.auth.resetPasswordForEmail(email.trim(), {
    redirectTo: `${WEB_URL}/auth/callback?next=/reset-password`,
  });
  setLoading(false);
  if (error) return Alert.alert("Hata", "E-posta gönderilemedi.");
  Alert.alert(
    "E-posta Gönderildi",
    "Şifre sıfırlama bağlantısı e-postana gönderildi. Linke tıkladıktan sonra şifreni web sitemizden değiştirebilirsin.",
    [{ text: "Tamam", onPress: () => router.back() }]
  );
};
```

**Düzenlenecek:** `mobile/app/(auth)/login.tsx`

Şifre input'u altına "Şifremi unuttum" linki ekle:
```tsx
<Link href="/(auth)/forgot-password" asChild>
  <TouchableOpacity className="self-end mb-4">
    <Text className="text-blue-500 text-sm">Şifremi unuttum</Text>
  </TouchableOpacity>
</Link>
```

**Not:** `EXPO_PUBLIC_WEB_URL` env değişkeni canlıya alındıktan sonra `.env` veya `app.json` `extra`'ya eklenecek (kullanıcı kararı).

**Düzenlenecek:** `mobile/app/(auth)/_layout.tsx` — Expo Router file-based otomatik tanır; manuel ekleme şart değil.

- Mesaj satırı tıklanınca: `router.push(\`/messages/\${row.chat_id}\`)`

## Adım 4: Mimari & Güvenlik Düzeltmeleri

### 4a. Bookmarks gizlilik düzeltmesi (yeni migration)

**Yeni dosya:** `supabase/migrations/023_bookmarks_privacy.sql`

```sql
-- Yer imleri sadece sahibi tarafından okunabilmeli
drop policy if exists "Yer imleri herkese açık" on public.bookmarks;
create policy "bookmarks_select_own"
  on public.bookmarks for select
  using (auth.uid() = user_id);
```

**Risk:** Web/mobil kodda "kullanıcının bookmark'larını join ile çekme" varsa sadece auth user için çalışacak (zaten doğru). Profil sayfasında "kayıtlı gönderilerim" tab'ı kendi profilinde olduğundan sorun yok.

### 4b. Schema duplicate temizliği

**Silinecek:** `web/src/lib/schemas.ts`
**Düzenlenecek:** Tüm import'ları (`from "@/lib/schemas"`) → `from "@errorlife/shared/schemas"` olarak güncelle.

Önce grep ile import sayısı/yerleri tespit edilecek. Workspace dependency çalışıyor zaten (`package.json`'da `"@errorlife/shared": "file:../packages/shared"`).

### 4c. Local `Bid` type duplicate temizliği

**Düzenlenecek:** `mobile/app/jobs/[id]/index.tsx` (satır 24-34)

Local `Bid` type'ı kaldır, `import type { Bid, BidWithExpert } from "@errorlife/shared/types"` kullan.

### 4d. SCOPE DIŞI bırakılan duplicate

`web/src/types/database.ts` ve `mobile/src/types/database.ts` Supabase autogenerated `Database` tipini içeriyor — composite type alias'ları kullanılıyor. Bunları shared'a taşımak için autogenerate workflow'u kurmak gerekir; ayrı bir task olarak ileriye bırakılıyor.
